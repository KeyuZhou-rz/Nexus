# Nexus Engineering Guardrail

## Product Vision (Must Not Drift)
Nexus 的最终目标是一个**一体式的、0摩擦的、聪明的学习助手**。

## Execution Principles
1. 优先实现端到端学习闭环：数据获取 -> 结构化记忆 -> 可行动建议。
2. 任何开发决策优先减少用户摩擦（减少手动步骤、减少重复配置）。
3. 优先提升“聪明度”相关能力：对话记忆、薄弱点识别、纠偏学习。
4. UI/校验/重构工作必须服务于主闭环，不应偏离主线。
5. 每轮开发都要回答：这项改动是否让用户更接近“无感使用+持续变聪明”？

## MVP Priority Order
1. 对话记忆与薄弱点结构化提取（核心）
2. 自动化数据摄入与失败恢复（稳定）
3. 文档按需引用与知识检索（增强）
4. UI优化与展示层细节（最后）
